# Security Policy

Perspecta welcomes responsible reports of security vulnerabilities. Please
report suspected vulnerabilities privately and avoid public disclosure until
the issue has been reviewed.

## Supported Versions

| Version | Supported |
| --- | --- |
| Latest published release | Yes |
| Older releases | No |

## Reporting a Vulnerability

Please use GitHub private vulnerability reporting through the
[Report a vulnerability](https://github.com/timcogan/perspecta/security)
flow in the repository Security tab.

Do not open a public GitHub issue for suspected vulnerabilities.

Please include the affected version, reproduction steps, impact, and any
suggested mitigation if known.

## What to Expect

Reports will be reviewed privately.
Remediation timing depends on severity, complexity, and release scheduling.
